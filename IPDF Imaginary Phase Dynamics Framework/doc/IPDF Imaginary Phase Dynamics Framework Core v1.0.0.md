# 虚数位相力学フレームワーク設計書（実装対応版）

**Imaginary Phase Dynamics Framework（IPDF）**

---

## 1. 概要

本フレームワークは、「未来の潜在分布（Latent Future Distribution）」を内部状態として保持し、現在の観測状態と統合することで意思決定を行う動的制御モデルである。

本モデルは、

・現在の観測状態（実数成分）  
・将来の確率分布（虚数成分）

を同時に扱い、不確実性を内包したまま行動選択を行う。

ここでの虚数とは数学的虚数ではなく、

> 未発生だが意思決定に影響を与える未来分布の内部表現（Latent Phase Component）

を意味する。

---

### 目的

・予兆に基づく先行制御  
・リスクの連続的減衰  
・不確実性を考慮した意思決定  
・行動継続性の維持  
・探索と安定の動的バランス  

---

## 2. 基本構造

### 2.1 拡張状態空間

システム状態は以下で表現される：

Z = (X, Y)

・X：現在の観測状態（環境・内部状態）  
・Y：未来の潜在確率分布（予測分布）  

---

### 2.2 虚数成分（未来分布）

Y は未来軌道または状態の確率分布として定義される：

Y = p(F | X)

F：未来状態系列（trajectory）

---

### 2.3 分布表現

Y は有限サンプル集合として近似される：

Y ≈ { (f_i, p_i) }

・f_i：未来軌道サンプル  
・p_i：重み（正規化済み）  

確率重みは：

p_i ∝ likelihood(f_i)^α

---

### 2.4 分布埋め込み（Distribution Embedding）

高次元分布 Y は制御・学習のために埋め込みベクトルへ変換される：

E_Y = Embed(Y)

例：

・統計モーメント（平均・分散・歪度）  
・セットエンコーダ（Deep Sets / Attention）  
・粒子特徴集約  

以降の関数 Q や制御は E_Y を入力とする。

---

### 2.5 派生指標

未来分布 Y から以下を導出する：

#### リスク

Y_risk = P(undesirable | Y)

#### 不確実性

Y_uncertainty = H(Y) / H_max

（H：エントロピー）

#### Optionality（選択柔軟性）

Y_optionality は「将来の選択余地の広さ」として定義される：

Y_optionality = w_E * ESS(Y)  
　　　　　　　+ w_D * Diversity(Y)  
　　　　　　　+ w_V * ValueSpread(Y)

・ESS(Y) = 1 / Σ p_i²（有効サンプル数）  
・Diversity(Y)：軌道間距離分散  
・ValueSpread(Y)：上位価値分布の分散  

※ w_E, w_D, w_V はタスク依存パラメータ

---

### 2.6 状態更新構造

Y は時間的に連続な内部状態として更新される：

Y_{t+1} = (1 - β) Y_t + β * Estimate(Futures_t)

β：更新率（0 < β ≤ 1）

---

## 3. システムパイプライン

観測 → 未来生成 → 分布推定 → 指標抽出 → 制御最適化 → 行動選択 → 状態更新

---

### 3.1 未来生成

現在状態 X から未来分布をサンプリング：

Futures = Generate(X, H, N)

・H：予測ホライゾン  
・N：サンプル数  

状態遷移：

X_{t+1} ~ P(X_{t+1} | X_t, a_t)

---

### 3.2 分布推定

サンプル集合から未来分布を構築：

Y ≈ { (f_i, p_i) }

低確率サンプル除去：

p_i < ε → 除去

---

### 3.3 計算制約

・Top-k選択  
・低確率除去  
・距離クリッピング  
・ローリングホライゾン  
・粒子フィルタ  

---

### 3.4 指標抽出

Y から以下を算出：

・Y_risk  
・Y_uncertainty  
・Y_optionality  

---

## 4. 制御構造

### 4.1 制御目的

行動選択は以下の最適化問題として定義する：

argmax_a  
E[Reward | Y, a]  
− λ_risk * Risk(Y, a)  
+ λ_opt * Optionality(Y, a)

---

### 4.2 制御重み

補助的に以下を定義：

R_weight = sigmoid(k_r * (Y_risk - θ_r))

D_weight = (1 - Y_risk) * Y_optionality

Speed = exp(-k_u * Y_uncertainty)

---

### 4.3 動的調整

D_weight = D_weight * Speed  
R_weight = R_weight + (1 - Speed)

---

### 4.4 正規化

D_weight + R_weight = 1

---

### 4.5 制約

0 ≤ D_weight ≤ 1  
0 ≤ R_weight ≤ 1  

|ΔX| ≤ δ_max  
|ΔY| ≤ γ_max  

---

## 5. 行動選択

Action は以下に従う：

Action ~ Softmax(Q(X, E_Y, a), τ)

・E_Y：分布埋め込み  
・τ：温度パラメータ  

---

### 制約

・極端行動の制限  
・最小行動量保証  

---

## 6. 状態更新

X_{t+1} = UpdateReal(X_t, Action)

Y_{t+1} = (1 - β) Y_t + β * Estimate(Futures_t)

---

## 7. 位相ダイナミクス

activity_level = |D_weight - R_weight|

phase = atan2(Y_risk, activity_level)

---

### 7.1 位相利用

・探索温度制御  
τ = τ_0 + τ_1 * sin(phase)

・学習率制御  
η = η_0 * (1 - Y_uncertainty)

・モード遷移制御  

---

### 7.2 モード定義

・探索モード（低リスク・高活動）  
・適応モード（中間状態）  
・安定モード（高リスク・低活動）

## 16. 設計思想

本フレームワークは予測装置ではない。

> 未来を当てるのではなく、未来分布を保持したまま現在を最適化する構造である

虚数成分は不確実性ではなく、

> 意思決定における制御資源である

---

## 17. 結論

本フレームワークは、

現在状態と未来分布を統合し、

> 未来を内包した現在の最適意思決定

を実現する。

これによりシステムは：

・予兆段階で適応し  
・リスクを連続的に抑制し  
・停止せず行動を継続する  

動的意思決定主体となる。
