# 周期ベロシティ力学フレームワーク（CVD Framework）設計書 v3.0

---

## 1. 概要

本設計書は、「周期ベロシティ力学（CVD）」を**プロダクトへ組み込むための汎用フレームワーク仕様**として再定義したものである。

本フレームワークは、以下の3要素を統合し、リアルタイム制御を可能にする：

* 状態（Drive / Rest）
* ベロシティ（0–10段階）
* 周期（時間構造）

さらに、本バージョンでは以下を追加する：

* API / インターフェース定義
* 入出力仕様
* アクションマッピング
* ステートマシン
* 外乱応答
* ログ / KPI / テスト設計

---

## 2. システム構成

```
[CVD Core Engine]
    ├── Time Manager（Δt更新）
    ├── Phase Calculator
    ├── Velocity Quantizer
    ├── State Resolver（D/R）
    ├── Disturbance Handler
    └── Action Mapper
```

## 4. API仕様

### 4.1 初期化

```text
init(config) -> CVDInstance
```

---

### 4.2 状態取得

```text
getState(t) -> {
  phase,
  state,           // D or R
  velocity_level,  // 0–10
  normalized_velocity
}
```

---

### 4.3 更新（リアルタイム）

```text
update(Δt) -> state
```

* フレーム単位で呼び出す
* Δtベースで時間進行

---

### 4.4 外乱適用

```text
applyDisturbance(event) -> state
```

---

### 4.5 パラメータ変更

```text
setCycle(T)
setAmplitude(A)
setPhase(φ)
```

---

## 5. 入出力仕様

### 入力

* Δt（時間差分）
* 外乱イベント
* ユーザー状態（任意）

---

### 出力

* 状態（D / R）
* ベロシティ（0–10）
* 位相（phase）
* 推奨アクション

---

## 6. ステートマシン

### 状態遷移

```
IDLE
 → R_LOW
 → R_HIGH
 → D_LOW
 → D_HIGH
 → IDLE
```

---

### 遷移条件

* 位相に基づく自動遷移
* 外乱による強制遷移あり

---

## 7. ベロシティモデル

### 正規化

[
V_n = \frac{|v(t)|}{V_{max}}
]

### 量子化

| レベル  | 範囲            |
| ---- | ------------- |
| 0    | 0.0           |
| 1–10 | 0.0–1.0 を10分割 |

---

## 8. アクションマッピング

### 基本ルール

| 状態 | ベロシティ | アクション    |
| -- | ----- | -------- |
| D  | 8–10  | 強調・拡張・加速 |
| D  | 4–6   | 通常表示     |
| R  | 7–9   | フェード・収束  |
| R  | 1–3   | 静止準備     |
| 任意 | 0     | 完全停止     |

---

### 出力形式

```json
{
  "action": "expand",
  "intensity": 0.8
}
```

---

## 9. 外乱応答モデル

### 9.1 外乱分類

| 種類 | 内容       |
| -- | -------- |
| 短期 | クリック・タップ |
| 長期 | モード変更    |
| 強制 | エラー・割り込み |

---

### 9.2 挙動

* 位相シフト発生
* 振幅変動
* ベロシティ急変

---

### 9.3 回復仕様

* 最大2周期以内に復帰
* ダンピングにより安定化

---

## 10. 時間処理モデル

### Δt更新式

```text
state(t + Δt) = f(state(t), Δt)
```

---

### 特性

* フレームレート非依存
* リアルタイム対応

---

## 11. マルチスケール対応

| レイヤー | 周期     |
| ---- | ------ |
| ミクロ  | 0.5–2秒 |
| メソ   | 5–30分  |

---

### ルール

* 上位周期が下位周期を制御
* 位相干渉を最小化

## 16. 設計原理

1. 周期性（すべては繰り返す）
2. 位相分離（状態と速度は独立）
3. 段階性（ベロシティは離散）
4. リズム最適化（最大化ではなく配分）

---

## 17. 制約

* 非周期現象には直接適用不可
* パラメータ調整が必要
* 外乱が多い環境では補正コスト増大

---

## 18. 結論

CVD Frameworkは、

> **周期・状態・ベロシティを統合したリアルタイム制御フレームワーク**

であり、

* UI
* 行動設計
* システム制御

において、**リズムベースの設計と実装を統一する基盤**となる。
