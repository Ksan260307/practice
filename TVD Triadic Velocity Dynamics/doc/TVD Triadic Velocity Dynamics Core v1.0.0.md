# Triadic Velocity Dynamics（TVD）設計書 v1.0.0

---

## 1. 概要

本設計は、三要素 **Drive / Regulation / Field（DRF）** による相関ダイナミクスを、プロダクトの中核ロジックとして長期運用・スケーリング・学習に耐える形で実装するための数理モデルである。

本モデルでは以下を実現する：

* 内部状態と外部環境の厳密分離
* 入力の安全な非線形処理
* 可観測性と制御指標の分離
* パラメータ管理の階層化
* 数値計算手法の明示
* ログおよびスケーリング前提の設計
* 数値積分時の発散リスクを排除したフェイルセーフ設計

---

## 2. 構造定義（DRF）

* **Drive（D）**：外向きの推進力
* **Regulation（R）**：内向きの制御
* **Field（F_internal）**：内部文脈状態
* **Field_external**：外部環境入力（正の値は圧力、負の値は緩和として機能）

---

## 3. 状態空間

* 内部状態：$D, R, F_{internal} \in [0, 1]$
* 外部入力：$F_{external} \in [-1, 1]$

---

## 4. Field統合

相互作用に使用するFieldは以下で定義：

$$F_{total}=\text{clamp}(F_{internal}+F_{external}, 0, 1)$$

---

## 5. 時間スケール

* $\Delta t$：固定（推奨 0.1）
* $\lambda_{eff}$：構造的安定性（中心への復元力）
* $\epsilon$：非線形復元力（高次安定化項。$\epsilon \ll \lambda_{eff}$）

---

## 6. 状態更新モデル

数値積分（RK2等）を適用するための連続的な常微分方程式（ODE）として定義する。
計算途中の境界外逸脱による発散を防ぐため、安定化項には線形復元力を採用し、定常状態が厳密に $0.5$ に収束するよう中心点からの偏差に対して非線形復元力を適用する。

### 6.1 基本式

直接作用ウェイト行列 $W_{direct}$ による入力の各次元成分を適用する。ここで使用する $\cdot$ はスカラーの積を意味する。

$$\frac{dD}{dt}=G(v_D)(F_{total}-R)-\lambda_{eff}(D-0.5)-\epsilon(D-0.5)|D-0.5|+\alpha_D \cdot \tanh((W_{direct}\cdot\text{Input})_D)$$

$$\frac{dR}{dt}=G(v_R)(D-F_{total})-\lambda_{eff}(R-0.5)-\epsilon(R-0.5)|R-0.5|+\alpha_R \cdot \tanh((W_{direct}\cdot\text{Input})_R)$$

$$\frac{dF_{internal}}{dt}=G(v_F)(D-R)-\lambda_{eff}(F_{internal}-0.5)-\epsilon(F_{internal}-0.5)|F_{internal}-0.5|+\alpha_F \cdot \tanh((W_{direct}\cdot\text{Input})_{F_{internal}})$$

※各項の絶対値表現 $|X-0.5|$ は、計算効率と実装の簡潔さを優先した硬化バネ項としての働きを持つ。

---

### 6.2 境界制約

RK2などの多段積分手法を用いる場合、中間の予測ベクトルを強制的に歪めて生じる不自然な振動や誤差の蓄積を防ぐため、毎ステップの積分完了後にのみ適用し、状態空間内を厳密に保証する。スケーリングやバッチ処理を前提とし、ベクトル操作（NumPy等）での実装を標準とする。

```python
import numpy as np

# 境界制約の適用（毎ステップの積分完了後に実行）
# D, R, F_internal をまとめたベクトル X に対する処理
X = np.clip(X, 0.0, 1.0)

# 必要に応じて個別の状態変数へアンパック
D, R, F_internal = X
```

## 7. ベロシティ設計

### 7.1 行列形式

$$X=[D, R, F_{internal}]^T$$

$$v=[v_D, v_R, v_F]^T$$

$$v=v_0+W_{state}X+W_{input}\text{Input}$$

---

### 7.2 正規化とスケーリング

理論モデルとしての正規化は標準シグモイド関数 $\sigma(x)=\frac{1}{1+e^{-x}}$ を用いる。

$$\tilde{v}_i=\sigma(v_i)$$

※実装上、ログ出力やUIでの可視化を行う場合は $v_{norm}=10\cdot\sigma(v)$ として $0 \sim 10$ のスケールに拡張して扱うことができるが、数理的なゲイン計算の入力としては上記の $\tilde{v}_i \in [0, 1]$ を直接用いる。

---

### 7.3 制約

* 対角優先
* スパース
* 正則化前提

---

## 8. ゲイン関数

各要素 $i \in \{D, R, F\}$ に対して個別に適用する。

$$G(v_i)=\exp(k\cdot(\tilde{v}_i-0.5))$$

これにより入力ゼロの定常状態（$\tilde{v}_i=0.5$）でゲインが $1.0$ となり、中心点が正確に補正される。

---

## 9. 入力設計

### 9.1 入力範囲

$\text{Input} \in [-1, 1]^N$
※ $N$ は入力次元数

---

### 9.2 非線形処理

状態直接作用。$N$ 次元の入力を各要素の作用量に変換するためウェイト行列 $W_{direct}$ を介する：

$$\Delta X_{input}=\alpha \circ \tanh(W_{direct} \cdot \text{Input})$$

※この $\Delta X_{input}$ は、セクション6.1のODEにおける各入力駆動項（第4項）のベクトル表現である。
※ $\alpha = [\alpha_D, \alpha_R, \alpha_F]^T$ は各次元への入力強度を制御する。
※ $\circ$ は要素ごとの積（アダマール積）を示す。

---

### 9.3 入力経路

* ベロシティ経路（構造変化・中長期的な傾向）
* 状態経路（即時反応・短期的なスパイク）

---

## 10. 観測レイヤー

### 10.1 状態観測

内部状態の正確なスナップショットを取得する。

```python
def observe_state(D, R, F_internal):
    return {
        "D": D,
        "R": R,
        "F_internal": F_internal
    }
```

---

### 10.2 制御観測

外部環境を含めた統合的なダイナミクス指標を算出する。超越関数（$\tanh$）による計算コストのボトルネックを解消するため、軽量な近似式 $f(x) = x / (1 + |x|)$ を標準として採用する。

※注意：この近似関数は $\tanh$ と比較して飽和域での値が小さくなる（例：$x=1$ のとき $\tanh(1) \approx 0.76$ に対し、近似式は $0.5$ となる）。後段のシステムで閾値判定やUI描画を行う場合は、このスケールの差異に留意すること。

```python
def observe_control(D, R, F_total, dD, dR, dF_internal):
    # tanhの高速近似関数を採用
    x = (D - R) * (1 + F_total)
    approx_output = x / (1 + abs(x))

    return {
        "drive": D - R,
        "pressure": F_total,
        "activity": D + R + F_total,
        "output": approx_output,
        "dD": dD,
        "dR": dR,
        "dF_internal": dF_internal
    }
```

## 11. 数値積分

### 11.1 デフォルト

* RK2（推奨・第6章のODEに対して適用）

```python
method = "RK2"
# 注1：中間ステップ（k1算出時等）での境界制約（クランプ）は、
# 境界付近での不自然な振動や誤差の蓄積を防ぐため行わない。
# 積分完了後の最終ステップでのみクランプを適用し、動的システムの滑らかさを維持する。

# 注2：1ステップ内のRK2計算中（中間ステップの評価を含む）は、
# 開始時点 t でサンプリングした Input の値を固定して使用し、再サンプリングは行わない。
# 外部環境からのフィードバック（F_external）が動的に変動するシステムの場合も同様に、
# 積分計算中は t 時点の固定値を使用し、再計算は行わない。
```

* Euler（計算リソースが極めて限定的な用途）

## 16. ドリフト・数値破綻対策

通常の境界制約（クランプ）では対応できない、NaNまたは無限大（Inf）の発生を検知した場合のフェイルセーフ処理。計算破綻時は安全なデフォルト値へのハードリセットを行う。
外部入力に起因する汚染を防ぐため、ベクトル処理を用いた積分計算前のサニタイズも必須とする。

```python
import numpy as np

# 1. 外部入力のサニタイズ（毎ステップ、積分計算の前に実行）
# 異常な入力はゼロとして無効化する
safe_input = np.where(np.isfinite(Input), Input, 0.0)

# --- RK2などの数値積分プロセス実行 ---

# 2. 内部状態のNaNおよびInfinity検知（毎ステップ、積分完了後に実行）
if not np.all(np.isfinite(X)):
    # 安全な状態（システム中心点）へ強制ハードリセット
    X = np.full_like(X, 0.5)
    D, R, F_internal = X
```

## 17. 初期化

ノイズによる境界逸脱を防ぐため、初期化時にも境界制約を適用する。ノイズは再現性を確保するため、シード管理された正規分布等からサンプリングする。

```python
import numpy as np

# σ_noise=0.05 程度の平均0の正規分布からのサンプリングを推奨
noise = np.random.normal(0, 0.05, 3)

X = np.clip(0.5 + noise, 0.0, 1.0)
D, R, F_internal = X
```

## 18. 設計原則

1. **分離性**：内部状態と外部環境を物理的・論理的に分離する。
2. **非線形性**：入力は必ず $\tanh$ 等の非線形変換を介し、系の飽和を防ぐ。
3. **多重減衰**：復元力（$\lambda_{eff}$）と非線形復元力（$\epsilon$）の役割を分離し、安定性を確保する。
4. **観測分離**：生の状態量と、アプリケーションが利用する制御指標を分離する。
5. **安定性優先**：精度と安定性が対立する場合、境界制約やハードリセットによる安定性を優先する。
6. **透過性**：全パラメータと中間変化量をログ出力可能にする。
7. **拡張性**：入力次元 $N$ や並列インスタンス数に依存しない構造を維持する。
8. **堅牢性**：数値計算の全ステップにおいて有限性と境界内であることを担保する。

## まとめ

TVD v1.0.0（DRFモデル）は、三要素相関ダイナミクスを、理論・実装・運用の全レイヤーで統合した完成系モデルである。

主な特徴：

* Drive / Regulation / Field の直感的な意味構造
* 外部環境 $F_{external}$ との安全な統合と定義
* 非対称ゲインおよび多点直接入力による高度な制御性
* 非線形復元力とRK2積分を組み合わせた、滑らかで安定した計算手法
* 入力サニタイズとベクトル演算による厳密な数値的安全性とスケーラビリティ
* スケーラブルかつ高精度な数値積分プロセス

本モデルは、抽象的な動的プロセスを実用的なプロダクトエンジンとして定着させるための強固な基盤を提供する。
