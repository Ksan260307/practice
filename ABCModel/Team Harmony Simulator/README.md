# Team Harmony Simulator

ABCモデル v3.2.0 に基づいた、**チーム内の人間関係と相互作用を可視化** するシミュレーションツール。個人の性格パラメータ（作業能力、基準の高さ、配慮、コンディション）がどのようにチーム調和度と離職リスクに影響するかを直感的に理解します。

## 概要

チームの仕事ぶりが「淡々と」に見える場合と「話しにくい」と感じられる場合の違いは何か？

個人の優秀さだけでは測れない、心理的な相互作用を数式化しシミュレートすることで、以下を学べます：

- 同じ高い成果でも、**配慮とコンディション次第でチーム評価が激変する**
- 「基準が高い」と「威圧的」は紙一重
- 心身の余裕がないと、いかに優秀でも周囲を不安にさせる

## 主要機能

### ABCモデルに基づく4つのパラメータ

#### A: Authenticity → 個人の作業能力
- **心理学的意味**: 仕事をこなす速度・正確性・自然な行動力
- **UI表示**: オレンジ色、😊のアイコン
- **スケール**: 0-100

**特徴**:
- 高い（80+）: 仕事は早く正確。ただし周囲への要求も厳しくなりがち
- 低い（20以下）: ミス・遅刻が増える。仕事ぶりが視界に入りにくくなる

#### B: Aesthetic → 理想・基準の高さ
- **心理学的意味**: 完璧主義、品質へのこだわり、妥協を許さない度合い
- **UI表示**: 紫色、 ⚠️ のアイコン
- **スケール**: 0-100

**特徴**:
- 高い（80+）: 品質を保証。ただし「妥協できない」印象が強くなり、話しかけにくくなる
- 余裕がないと「威圧」に変わる

#### C: Meta → 相手への配慮・俯瞰力
- **心理学的意味**: 周囲への気遣い、チーム全体を見る視点、調整能力
- **UI表示**: 青色、 👥 のアイコン
- **スケール**: 0-100

**特徴**:
- **チームを円滑にする最大の鍵**
- 高い（80+）: 「話しやすい」「相談しやすい」という信頼が生まれる
- 低い（20以下）: 作業は完璧でも「話しにくい」という苦情につながる

#### E: External → 心身のコンディション
- **心理学的意味**: 睡眠、体調、ストレス、心の余裕
- **UI表示**: 赤色、 💗 のアイコン
- **スケール**: 0-100

**特徴**:
- 高い（80+）: 余裕が表情に出て、周囲も安心
- 低い（20以下）: つい「ため息」が出て、周囲を不安にさせる

### シミュレーション指標

#### チーム調和度（0-100%）
```
調和度 = max(0, min(100, (C × 0.6 + E × 0.4) - (B - A) × 0.2))
```

**意味**:
- 配慮（C）と余裕（E）が高いほど調和
- 基準（B）が作業能力（A）より高すぎると悪化
- チーム内での信頼度・相談しやすさを表現

#### 離職・解約リスク（0-100%）
```
リスク = min(100, 破綻要因 + (100 - 調和度) × 0.5)
```

**破綻要因**:
- B=高 ∧ C=低 → +40 （基準高・配慮低の組み合わせ）
- E < 40 → +30 （心身の余裕がない）
- A=高 → +10 （仕事ができる故の「だるさ」感）

### 3段階のステータス表示

| ステータス | リスク | メッセージ | 記述 |
|-----------|--------|-----------|------|
| **良好** | < 40% | 「淡々と仕事をこなしています」 | 🤓 青 |
| **注意：とっつきにくい** | 40-70% | 「作業は完璧ですが、周囲は話しかけるのをためらっています」 | 😥 黄 |
| **危険：契約終了の危機** | > 70% | 「『話しにくい』という苦情が殺到しています」 | ⚠️ 赤 |

### ビジュアルフィードバック

- **トップでプログレスバー**: リスク率に応じて色が変わる（左から青→黄→赤）
- **中央の大きいメッセージ**: キャラの状態とセリフ
- **スコアボード**: 調和度とリスクの数値表示
- **リアルタイム更新**: スライダー変更に即座に反応

## 使用方法

### 基本操作

1. **4つのスライダーをドラッグして調整**

   - リアルタイムでシミュレーション結果が更新

2. **結果を観察**
   - 上部のプログレスバーが色変わり
   - 中央のメッセージが切り替わる

3. **インサイトを確認**

### インタラクティブな実験

#### 実験1: 完璧主義者層（基準が高く配慮が低い）
```
A: 90  （作業能力 高い）
B: 90  （基準 高い）
C: 20  （配慮 低い）
E: 50  （コンディション 普通）

結果: リスク 65% 以上（注意～危険）
メッセージ: 「作業は完璧ですが、周囲は話しかけるのをためらっています」
```

→ **優秀でも「話しにくい」という苦情が生まれる典型例**

#### 実験2: バランス型（各要素が中程度）
```
A: 70
B: 60
C: 70
E: 70

結果: リスク 低い（20%未満）
メッセージ: 「淡々と仕事をこなしています」
```

→ **完璧さより、バランスがチームを安定させる**

#### 実験3: 配慮不足型からの回復
```
初期状態:
A: 85, B: 80, C: 15, E: 40
→ リスク 70%+ （危険）

改善:
C を 15 → 60（配慮を上げる）のみ変更
→ リスク 40%へ低下

さらに:
E を 40 → 80（コンディション改善）
→ リスク 20%以下：良好状態に
```

→ **コンディション改善と配慮UPで劇的に改善**

#### 実験4: 疲弊状態の悪化
```
A: 90, B: 85, C: 50, E: 20
→ リスク 70%+ （危険：契約終了の危機）

理由: 
- 作業能力と基準は高い（仕事は完璧に見える）
- だが心身の余裕がなく表情に出ている
- 「このスタッフ、大丈夫？」という不安感
```

→ **能力が高いほど、コンディション悪化の影響は大きくなる**

## 計算式詳細

### 調和度の計算

```javascript
harmony = Math.max(0, Math.min(100, 
  (paramC * 0.6 + paramE * 0.4) - (paramB - paramA) * 0.2
));
```

**要素の意味**:
- **C × 0.6**: 配慮が調和度の最大要因（60%）
- **E × 0.4**: コンディション（40%）
- **(B - A) × 0.2**: 基準が能力を超える分がマイナス影響
  - A > B なら正の補正（余裕がある印象）
  - A < B なら負の補正（無理してる印象）

### リスク計算ロジック

```javascript
let ruinProb = 0;
if (vB === 2 && vC === 0) ruinProb += 40;  // 基準高・配慮低
if (vE < 0.4) ruinProb += 30;              // 余裕なし
if (vA === 2) ruinProb += 10;              // 仕事ができる故の複雑さ

const finalRuin = Math.min(100, ruinProb + (100 - harmony) * 0.5);
```

**ロジック**:
1. 破綻要因を検出（B高・C低の組み合わせは+40の大ペナルティ）
2. コンディション悪化を追加
3. 調和度の不足をさらに加算

## パラメータ構造

```javascript
// UI入力（0-100）
paramA: number,  // 作業能力
paramB: number,  // 基準の高さ
paramC: number,  // 配慮・俯瞰
paramE: number,  // コンディション

// 内部計算（0/1/2スケール）
vA = paramA > 70 ? 2 : paramA > 30 ? 1 : 0;
vB = paramB > 70 ? 2 : paramB > 30 ? 1 : 0;
vC = paramC > 70 ? 2 : paramC > 30 ? 1 : 0;
vE = paramE / 100;  // 連続値 0.0-1.0

// 出力
{
  harmony: 0-100,  // チーム調和度
  ruin: 0-100,     // 離職リスク
  statusLabel: string,
  message: string,
  color: string,   // Tailwind bg-color
  emoji: string
}
```

## 実務的な解釈

このシミュレーターから導かれる**組織マネジメントの洞察**：

### 1. 能力と人間関係は別軸

優秀なスタッフでも「話しにくい」という評価を受けることがあります。これは本来、**別の軸の問題** です。

### 2. 基準（B）は必要だが、配慮（C）がないと逆効果

- B 高 + C 高 = 信頼できるしっかり者
- B 高 + C 低 = 話しかけづらい完璧主義者
- B 低 + C 高 = 優しいがルーズ
- B 低 + C 低 = 頼りにならない

### 3. コンディション（E）は見落とされやすい

心身の余裕がないと：
- 表情が暗くなり周囲を不安にさせる
- つい「ため息」や「物言い」がきつくなる
- パフォーマンスは保ちやすいが雰囲気が悪化

### 4. 配慮（C）は最もスケーラブル

- 能力（A）: 育成に時間がかかる
- 基準（B）: 元々の気質が大きい
- 配慮（C）: 意識的に高められる
- コンディション（E）: 制度や環境で改善できる

→ **配慮とコンディションの改善が最も効果的**

## 応用例

- **人材評価**: 「仕事ぶり」だけでなく「チームへの貢献」を数値化
- **組織開発**: メンバーの雰囲気悪化の原因をデータ化
- **1on1 面談**: 「話しにくい」という部下からの評判改善策を可視化
- **マネジメント教育**: 心理的安全性の実現方法を直感的に理解
- **キャリア相談**: 個人のキャリアと組織への貢献のバランス検証

## 注記

- このシミュレーターはモデル化された簡略版です。実際の人間関係はさらに複雑です
- パラメータは個人差があり、このモデルが全てではありません
- 目的は「完璧な測定」ではなく「観点の提供」です
- 実際の組織判断には、定性的なコミュニケーションが必須です
