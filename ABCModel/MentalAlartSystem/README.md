# Mental Alert System

複雑な心理状態をシミュレートする高度な診断・監視システム。心の状態をリアルタイムで追跡し、破綻リスク（RuinScore）を段階的に評価します。

## 概要

Mental Alert Systemは、ABCモデル（Authenticity・Aesthetic・Meta）と環境ストレス要因（EF）の相互作用を物理シミュレーターのように計算し、心理的破綻の危険性を予測・監視します。

従来の静的診断ではなく、時間軸上で心理状態がどう変化するかを**動的にモデル化**する点が特徴です。

## 主要機能

### コアシステム（ABCモデル）

各コアは3つのパラメータで表現：

#### A: Authenticity（行動力・モチベーション）
- **v（速度/強度）**: 0-2（エネルギー量）
- **Δ（デルタ/揺れ）**: 0-2（安定性）
- **state**: Normal / Zero / Runaway

#### B: Aesthetic（こだわり・責任感）
- **v（速度/強度）**: 0-2（基準への執着）
- **Δ（デルタ/揺れ）**: 0-2（基準のブレ）
- **state**: Normal / Zero / Runaway

#### C: Meta（客観性・気配り）
- **v（速度/強度）**: 0-2（監視レベル）
- **Δ（デルタ/揺れ）**: 0-2（監視の激しさ）
- **state**: Normal / Zero / Runaway

### 環境ストレス要因（EF: Environmental Factors）

| ラベル | 説明 | 入力レベル |
|--------|------|-----------|
| E0 | 体調・睡眠 | -1（虚無）/ 0（無）/ 1（少）/ 2（中）/ 3（強） |
| E1 | 物理環境ストレス | 0-3 |
| E2 | 社会圧・評価 | 0-3 |
| E3 | タスク過多・雑音 | 0-3 |

### シミュレーションロジック

#### ステップ1: 外部要因の投影
EFコンポーネントがコアのv・Δ・stateに影響を加えます。

```
A: E1, E3, E0 によりvが低下
B: E2（社会圧）と E3 によりv上昇
C: E2, E3, E0 によりv上昇
```

#### ステップ2: コア間の相互作用
- **A→B**: 行動力がプレッシャーを制御（減衰）
- **B→C**: こだわりが自己監視を強化
- **C→A**: 過度な監視がエネルギーを喪失させる

#### ステップ3: トリガールール（TR）

システムが自動的に検出する危険パターン：

| ルール | 条件 | 結果 |
|--------|------|------|
| **TR1** | C暴走＆A虚無 | A→虚無ロック |
| **TR2** | A虚無＆B暴走＆B強い | C暴走 |
| **TR5** | E2(社会圧)暴走 | B暴走 |

#### ステップ4: Zero-Lock管理

「虚無ロック」状態でAが虚無化されると：
- 追加のストレス（E0, E2, E1）でロックが深化
- 暴走状態が2ステップ連続して無いと、少しずつ解除

#### ステップ5: RuinScore計算

各コアと各EFについてスコア（0-6）を計算し、最大値を採用。

スコア範囲：
- **0-2**: 安定
- **3-4**: 注意
- **5**: 危険
- **6**: 破綻

### プリセット機能

以下の 4 つのシナリオが用意されています：

| プリセット | 説明 | EF設定 |
|-----------|------|--------|
| **overwork** | 過労状態 | E0=2, E1=3, E2=1, E3=2 |
| **sns_burnout** | SNS疲弊 | E0=1, E1=0, E2=3, E3=3 |
| **sick** | 体調不良（虚無） | E0=-1, E1=1, E2=0, E3=0 |
| **rest** | 完全休息 | E0=0, E1=0, E2=0, E3=0 |

## 使用方法

### 基本操作

1. **環境ストレス設定**: E0-E3のスライダーで外部要因を設定
   - -1 から 3 の範囲で調整可能
   - -1 は「虚無体調」（特殊な悪化状態）

2. **プリセット適用**: 「Overwork」「SNS Burnout」などのシナリオボタンをクリック

3. **シミュレーション実行**: 
   - 「▶ 次のステップ」ボタンで1日進める
   - 日ごとに心理状態が変化
   - ログウィンドウに警告が記録される

4. **状態確認**:
   - **スコアバー**: RuinScoreの色分け表示
   - **コンポーネント**: A、B、Cそれぞれの v（速度）と Δ（揺れ）
   - **履歴ログ**: トリガー発動や状態変化を時系列で表示

5. **リセット**: 「🔄 システムリセット」でシミュレーション初期化

### UIコンポーネント

#### ComponentBar
各コア（A、B、C）の詳細情報をバー表示
- 状態バッジ（安定/虚無/暴走）
- 強度（v）プログレスバー
- 揺れ（Δ）プログレスバー

#### スコア表示
- **Green (0-2)**: 安定状態
- **Yellow (3-4)**: 注意状態
- **Orange (5)**: 危険状態
- **Red (6)**: 破綻状態

## パラメータ構造

```javascript
// コアの状態
{
  A/B/C: {
    v: 0-2,         // 速度/強度
    delta: 0-2,     // 揺れ/変動性
    state: 0|1|2    // STATE.NORMAL | ZERO | RUNAWAY
  }
}

// パラメータ管理
{
  zeroLock: 0-2,             // 虚無ロックの深さ
  runawayAbsentCount: 0+,    // 暴走が無かった連続日数
  cooldowns: {
    TR1: 0+,  // TR1トリガーのクールダウン
    TR2: 0+,  // TR2トリガーのクールダウン
    TR5: 0+   // TR5トリガーのクールダウン
  }
}

// スコア
{
  label: 0|2|4|5|6,  // 段階化されたRuinScore
  raw: 0-6           // 連続値の真のスコア
}
```

## ロジック詳細

### RuinScoreの計算公式

```
個別スコア = w_v * v + delta + state

ここで：
- w_v = -1.0 (Aの場合) または 1.0 (B,Cの場合)
- state = 0 (Normal) | 1 (Zero) | 2 (Runaway)
- スパイク状態（delta≧1.5 且つ Runaway）なら最小5.0
- Zero状態なら最小3.0

最終RuinScore = max(core全体, EF全体) をroundRBand()で段階化
```

### Del​taバンド判定

```
getDeltaBand(d):
  d < 0.5  → STABLE (0)
  0.5 ≤ d < 1.5 → SWING (1)
  d ≥ 1.5 → SPIKE (2)
```

### スコア段階化

```
roundRBand(r):
  r < 2.5 → 2 (安定)
  2.5 ≤ r < 4.4 → 4 (注意)
  4.4 ≤ r < 5.4 → 5 (危険)
  r ≥ 5.4 → 6 (破綻)
```
