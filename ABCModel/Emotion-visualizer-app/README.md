# Emotion Visualizer App（感情ビジュアライザー アプリ）

**テキストに隠れた感情を、見やすく、わかりやすく、やさしく可視化する。** 入力したことばの中から感情を自動分析し、複雑な心の動きを3層に分けて表現するインタラクティブツールです。

## 概要

これは、**認知行動療法（ABC）モデル** に基づいた感情分析システムとして設計されました。

プレイヤーは：
1. **テキストを入力** （短文だけでなく、日記や独り言でもOK）
2. **10個の感情を自動検出**（喜び・悲しみ・怒り・不安・驚き・嫌悪・期待・信頼・軽蔑・疲れ）
3. **6つのビジュアルパネル**で、感情の割合・分類・流れが即座に理解できる
4. **ABCフロー（きっかけ→受け止め方→気持ち）** を原文から抽出して表示

という体験を通じて、自分や他者の感情を **客観的かつ同情的に** 理解できます。

## 主要機能

### 1. 10個の感情カテゴリー

| 感情 | カラー | タイプ | 説明 |
|------|--------|--------|------|
| **喜び（Joy）** | 橙 🟠 | ポジティブ | 嬉しい、楽しい、安心、満足、感謝など |
| **悲しみ（Sadness）** | 紺 🟦 | ネガティブ | つらい、寂しい、落ち込み、失望、無力感など |
| **怒り（Anger）** | 赤 🔴 | ネガティブ | 腹が立つ、イライラ、むかつく、悔しい、許せないなど |
| **不安（Anxiety）** | 紫 🟣 | ネガティブ | 不安、心配、怖い、恐れ、焦り、そわそわなど |
| **驚き（Surprise）** | 青緑 🔵 | ニュートラル | 驚く、びっくり、意外、当惑、戸惑いなど |
| **嫌悪（Disgust）** | 水色 🟢 | ネガティブ | 嫌、嫌悪、うんざり、気持ち悪い、不快など |
| **期待（Expectation）** | 橙黄 🟡 | ポジティブ | 楽しみ、期待、ワクワク、知りたい、希望など |
| **信頼（Trust）** | 緑 🟩 | ポジティブ | 信頼、頼れる、安心感、尊敬、絆など |
| **軽蔑（Contempt）** | 紫 🟪 | ネガティブ | 見下す、軽蔑、侮辱、冷笑、あきれるなど |
| **疲れ・だるさ（Fatigue）** | グレー ⬜ | ネガティブ | 疲れ、だるい、倦怠、眠い、へとへと、無気力など |

### 2. 3層の感情分類（L0）

テキストから検出した感情を、3つの異なるレイヤーで分類：

#### 考えの感情（Cognitive emotions）
- **定義**: 判断・心配・期待などの頭の中の動き
- **含まれる**: 喜び、悲しみ、怒り、期待、信頼、軽蔑、驚き
- **検出ロジック**: 認知用語（「と思う」「気がする」「かもしれない」「べき」「無理」など）の出現数 + 主要感情の重み付け

#### 身体的な感情（Somatic emotions）
- **定義**: 疲れ・だるさ・緊張など体に出るサイン
- **含まれる**: 疲れ・だるさ、不安の一部
- **検出ロジック**: 身体用語（「胸」「胃」「肩」「手汗」「震え」「重い」など）の出現数 + 疲れのスコア

#### 混ざった感情（Mixed emotions）
- **定義**: 焦りや戸惑いなど、複数の側面が同時に出ている
- **含まれる**: 曖昧感、複雑さ、矛盾する感情の共存
- **検出ロジック**: 混在用語（「焦り」「戸惑」「複雑」「そわそわ」など）+ 不確実性指数 + ポジティブとネガティブの同時出現

### 3. スコアリングロジック

#### 基本スコア（Raw Scores）
各感情について、その感情を表す**エイリアス語彙リスト**と一致する箇所をカウント。

例：「喜び」の場合、35個のエイリアス語
```javascript
["嬉しい" (3.2重み), "うれしい" (3.2), "楽しい" (3.1), ..., "感謝" (2.4), ...]
```

各キーワードの出現回数 × 重み = その感情への寄与度

#### 否定句による修正
`否定詞 + 感情語` の場合、スコアを調整：

**ポジティブ感情の否定**（「喜びがない」など）:
- 該当感情のスコアを 55%減
- 悲しみ +0.9、不安 +0.7

**ネガティブ感情の否定**（「悲しくない」など）:
- 該当感情のスコアを 35%減
- 信頼 +0.5、喜び +0.35

#### Layer 0（3層）の計算
```
cognitiveSignal = 認知用語の合計 + (主要感情スコア × 0.26)
somaticSignal = 身体用語の合計 + (疲れスコア × 0.95)
mixedSignal = 混在用語の合計 + (不確実性 × 1.2) + (ポジティブ > 0 AND ネガティブ > 0 ? 2.5 : 0)

L0_percents = softmax([cognitiveSignal, somaticSignal, mixedSignal], temperature=1.18)
```

#### 最終感情スコア（正規化）
Top 5 感情を抽出し、合計 100%に正規化。端数処理で、合計が必ず 100% になるよう調整。

### 4. 特殊なシグナル検出

#### 皮肉・サルカズム（Sarcasm）
```
sarcasmMarkers = ["皮肉", "笑", "w", "最高", "さすが", ...]

if (sarcasmMarkers.count >= 1 || (positiveScore > 4 AND negativeScore > 5)) {
  sarcasm = true
}
```
表向きはポジティブだが実際にはネガティブ、という文脈を追捉。

#### 気持ちの強さ（Intensity）
```
intensityScore = negativeScore + positiveScore + mixedSignal + physioSignal + (emphasis × 0.8)

if (intensityScore < 5) intensity = "弱い"
if (5 <= intensityScore < 9) intensity = "やや弱い"
if (9 <= intensityScore < 14) intensity = "中程度"
if (14 <= intensityScore < 19) intensity = "やや強い"
if (intensityScore >= 19) intensity = "強い"
```

#### 傾向（Tendency）
```
positiveScore = joy + expectation + trust
negativeScore = sadness + anger + anxiety + disgust + contempt + fatigue

if (positiveScore > negativeScore × 1.15) tendency = "ポジティブ"
if (negativeScore > positiveScore × 1.1) tendency = "ネガティブ"
else tendency = "ニュートラル"
```

### 5. ABCモデル抽出（感情の流れ）

**A（Activating event）**：出来事・きっかけ
```
eventPatterns = [
  /.*?(言われた|された|起きた|あった|失敗した|うまくいかない).*?/g,
  /.*?(評価されない|間に合わない|疲れた).*?/g,
]

event = 最初にマッチした句 or 最初の文節
```

**B（Belief）**：受け止め方・解釈
```
beliefPatterns = [
  /.*?(べき|無理|だめ|失敗|足りない|嫌われ|心配|不安).*?/g,
  /.*?(と思う|気がする|かもしれない).*?/g,
]

beliefs = マッチした全句をリスト化（最大2件）
```

**C（Consequence）**：出てきた気持ち
```
feelings = topEmotions[0:3].map(emotion => emotion.label)
```

### 6. キーフレーズ抽出

3つの方法で、文章中の注目フレーズを抽出：

1. **コンテキストスニペット**: 感情語の前後 6文字を抽出
2. **節ベースのスコアリング**: 各文節に対して感情語数 + 認知用語数を計算
3. **高スコア句の抽出**: 上位3つをユニークで返す

結果として、「この文章のここが感情につながっている」という部分が可視化される。

### 7. サマリー生成

3行構成で、テキスト全体の感情の流れをやさしい言葉で説明：

**Line 1**: 主要感情と副感情の関係
```
例：「中心にあるのは『悲しみ』で、『疲れ』も一緒に動いています。」
```

**Line 2**: Layer 0 の特性を説明
```
例：「頭の中の考えより、体の反応やエネルギーの落ち方が強く出ています。」
```

**Line 3**: ABCの文脈から得た洞察
```
例：「『評価されない』をきっかけに、『自分はだめなのかもしれない』という受け止め方が気持ちの動きにつながっています。」
```

## 使用方法

### 基本的な操作

1. **テキスト入力**
   - メインの textarea に文章を入力
   - または、下部の「例文 1-4」ボタンをクリック
   - リアルタイムで分析が実行される

2. **結果の表示（6つのセクション）**

   **1. 入力文**
   - 原文をそのまま表示
   - 分析対象のテキストを確認

   **2. 感情の割合（主な気持ち）**
   - ドーナツチャート + 割合リスト
   - 最大5種類までの感情を、色付けして表示

   **3. 感情タイプの分類（3つの性質）**
   - 考え寄り / 体寄り / 混ざり具合
   - 合計 100% で表示

   **4. 感情の簡易まとめ**
   - 専門用語を使わず、やさしい言葉で整理
   - 3行の平易な説明

   **5. 文章中のポイント表示**
   - 感情が表れやすい箇所を原文から抽出
   - 最大3つのフレーズ

   **6. 追加情報**
   - 皮肉の有無
   - 気持ちの強さ
   - 全体としての傾向

3. **右パネル（ABCフロー）**
   - きっかけ（出来事）
   - 頭の中で起きたこと（受け止め方）
   - 出てきた気持ち（感情）
   
   3つの柱立てで、感情の因果関係を視覚化

### インタラクティブな実験

#### 実験1：「否定句による感情反転」
```
入力 (A): "楽しい"
結果: 喜び 100%

入力 (B): "楽しくない"
結果: 悲しみ ↑, 喜び ↓
→ 完全に反転せず、若干の曖昧性を保持
```

#### 実験2：「複数感情の同時出現」
```
入力: "頑張ったのに失敗した。悔しいし疲れた。でも次はやってみたい。"

期待結果:
- 悔しさ（怒り）: 35%
- 疲れ・だるさ: 30%
- 期待: 25%
- 悲しみ: 10%
- 傾向: ニュートラル（ポジティブとネガティブが混在）
- Layer 0: 混ざった感情が高い
```

#### 実験3：「皮肉の検出」
```
入力 (A): "あの人、本当に最高だね。笑"
入力 (B): "あの人、本当に最高だね。"

結果:
(A) sarcasm: ありの可能性 → 実際にはネガティブ方向にシフト
(B) sarcasm: 目立たない → そのままポジティブ
```

#### 実験4：「ABCフローの抽出」
```
入力: "明日の発表が近づいてきた。うまくいくか不安だし、話し方が下手だったらどうしようと思う。でも頑張ればできるかもしれない。"

期待結果:
- きっかけ: "明日の発表が近づいてきた"
- 頭の中: "うまくいくか不安だし", "どうしよう"
- 出てきた気持ち: 不安 / 期待 / 信頼
```

#### 実験5：「身体的な感情の検出」
```
入力1: "疲れた"
入力2: "疲れた。胸が痛い。息が苦しい。"
入力3: "疲れた。でも気持ちは前向き。"

結果:
- 入力1: 疲れ 100% / Layer 0: 身体的 (高)
- 入力2: 疲れ + 心理的不調 + 身体症状
- 入力3: 疲れ + 喜び / Layer 0: 混ざった感情
```

## 心理学的背景

このシステムは以下の概念を統合しています：

### ABCモデル（認知行動療法）
- **A（Activating Event）**: 出来事
- **B（Belief）**: その出来事に対する受け止め方・解釈
- **C（Consequence）**: 結果として生じる感情・行動

Albert Ellis の理論に基づき、感情は出来事そのものではなく、その **解釈** によって生じることを表現。

### 感情の階層化
- **基本感情**: Ekman の 6基本感情 + 複合感情
- **認知的感情**: 思考と結びついたもの
- **身体的感情**: 生理的反応を伴うもの
- **混合感情**: 複数の側面が同時に出現

### NLPテキスト分析
- **キーワード抽出**: 感情語彙データベースの照合
- **否定処理**: 言語的修飾子の影響を計算
- **文脈理解**: 簡易的な句構造解析

## 応用例

- **セルフ・エクスプロレーション**: 日記の感情分析
- **コミュニケーション支援**: 相手の表現の背景にある感情を理解
- **心理教育**: ABCモデルの実践的な学習
- **カウンセリング補助**: セッション中の気持ちの整理
- **ウェルネスチェック**: 日々の感情の傾向把握
- **創作支援**: キャラクターの感情描写の参考

## 限界と注記

- このアプリは **参考ツール** であり、医学的診断ではありません
- 短い検出キーワード（「嫌」など）は複数の感情に重複計上される可能性があります
- 文化・方言による表現の多様性には完全には対応していません
- 複雑な文脈や文学的表現の解釈にはデータが限定的です
- 皮肉やジョークの判定は確実ではなく、参考値です
- 感情は連続的であり、10個のカテゴリーで完全に表現できない場合があります

## 拡張と API 実装

### 現状
このアプリは **完全にクライアント側で動作**しており、サーバーへの通信は不要です。すべての感情分析が React Hooks 内で実行され、リアルタイムで結果が画面に表示されます。

### サーバー側実装のメリット

1. **より高度な NLP**: 形態素解析、依存構造解析での精度向上

3. **機械学習モデル**: BERT、GPT などの大規模言語モデルの活用
4. **キャッシング**: よく分析される文章の結果をキャッシュして高速化
5. **カスタマイズ**: 特定のドメイン・言語モデルの利用
6. **統計**: ユーザーの分析履歴を収集して傾向分析

### クライアント側でのサーバー API 呼び出し（改造例）

```javascript
async function analyzeTextWithServer(text) {
  try {
    const response = await fetch('/api/analyze-emotion', {
      method: 'POST',

      body: JSON.stringify({

        model: 'advanced',

    if (!response.ok) {
      throw new Error(`API error: ${response.statusText}`);
    }

    const { data } = await response.json();
    return data;  // サーバーから返された分析結果を使用
  } catch (error) {
    console.error('Analysis failed:', error);
    // フォールバック：クライアント側の分析を使用
    return analyzeText(text);  // 従来のクライアント側実装
  }
}
```

### 段階的な実装戦略

1. **Phase 1（現在）**: クライアント側のみで運用
   - 導入が簡単、サーバーが不要
   - 完全オフライン対応

2. **Phase 2（オプション）**: サーバー API を追加
   - より高精度な分析が可能
   - 統計情報の収集

   - Google / Azure / OpenAI などを活用

### トラブルシューティング

**API が遅い場合:**
- レスポンスをキャッシュする
- バッチ処理で複数テキストを一度に分析
- 非同期処理（Web Worker）の活用

**精度が不安定な場合:**
- モデルの fine-tuning を調べる
- 訓練データの偏りを確認
- ハイパーパラメータの調整

**コスト削減の場合:**
- ローカル機械学習モデル（PyTorch, TensorFlow）の活用
- API のキャッシング機能
- リクエスト数の制限や割り当て管理
