# Robot Tuning Simulator

ABCモデルを**ロボット制御システム** のメタファーで表現する、インタラクティブなシミュレーター。複雑な心理力学を、機械的な相互作用として可視化し、パラメータチューニングを通じてシステムの安定性を理解します。

## 概要

心理学の教科書的な説明は難しく、数字だけでは実感が湧きません。このシミュレーターでは、ABCモデルの各軸を**ロボット内部システム** として具体化することで：

- **A（出力）**: ジェネレーター（動力源）
- **B（制御）**: スタビライザー（制御システム）
- **C（感知）**: 戦術AI（センサー・推論システム）
- **E（環境）**: 外部負荷（外的ストレス）

という対応付けで、システムの相互作用を直感的に理解できます。

## 主要機能

### 4つのシステムパラメータ

#### A: ジェネレーター出力（Authenticity）
- **物理学的イメージ**: ロボットの動力源、エネルギー出力
- **心理学的意味**: 個人の行動力、体感、自然な反応
- **スケール**: 0-2.0
- **表示**: 機体中枢の色と輝き

**動作**:
- 高い（1.5以上）: エネルギッシュ、安定稼働
- 適切（0.5-1.5）: バランスの取れた出力
- 低い（0.3以下）: システムダウン（Zero状態）

#### B: スタビライザー制御（Aesthetic）
- **物理学的イメージ**: ロボットの制御回路、防御装甲
- **心理学的意味**: 基準・こだわり、対外的な厳密性
- **スケール**: 0-2.0
- **表示**: 装甲・腕・脚の色と動き

**動作**:
- 高い（1.6以上）: 制御が過剰に働く（Runaway状態）
- 適切（0.5-1.5）: 安定した制御
- 低い（0以下）: 制御が弱すぎ

#### C: 戦術AIセンサー（Meta）
- **物理学的イメージ**: ロボットの頭部、カメラ、推論エンジン
- **心理学的意味**: 俯瞰力、他者への配慮、社会認識
- **スケール**: 0-2.0
- **表示**: ロボットの頭部のアンテナと視線

**動作**:
- 高い（1.6以上）: 過度な監視・自己意識（Runaway状態）
- 適切（0.3-1.5）: バランスの取れた認知
- 低い（0.3以下）: 周囲への無関心

#### E: 外部環境負荷（Environment）
- **物理学的イメージ**: 外部からのストレス・負荷・温度上昇
- **心理学的意味**: 体調不良、作業過多、人間関係ストレス
- **スケール**: 0-1.0

**動作**:
- すべてのシステムに悪影響を与える
- A を低下させ、B と C を上昇させる

### システム状態（State）

各パラメータは3つの状態を持ちます：

| 状態 | 値 | 記号 | 意味 |
|------|-----|------|------|
| **Normal** | 0 | ⚙️ | 正常稼働 |
| **Zero** | 1 | ⏹️ | システムダウン（v ≤ 0.3） |
| **Runaway** | 2 | 🔥 | 暴走状態（v ≥ 1.6） |

### 機体崩壊危険度（RuinScore）

0～6の整数スケール：

| スコア | 状態 | 説明 |
|--------|------|------|
| 0-2 | 🟢 安全 | システム正常、すべての機能が適切に動作 |
| 2-4 | 🟡 注意 | 異常兆候が現れ始めている |
| 4-5.9 | 🟠 危険 | 複数のシステムに異常、制御 が困難 |
| 5-6 | 🔴 危機 | 機体崩壊が目前、直ちの対応が必要 |

### 相互作用ルール（TR）

複数システム間の相互干渉を表現：

#### TR-1: 戦術AI過剰防衛
```
条件: C が Runaway ∧ A が低い
結果: A が Zero（強制停止）
意味: 過度な自己監視がエネルギーを奪い、システムダウン
```

実世界の例：「失敗を極度に恐れて行動停止」

#### TR-2: 制御干渉パニック
```
条件: B が Runaway ∧ A が Zero
結果: C が Runaway
意味: 制御系の過剰反応が、AIを暴走させる
```

実世界の例：「無理やり動かされてパニック状態に」

#### TR-3: 内なる光による安定化
```
条件: A が High（≥1.5）∧ B が Swing
結果: B が Normal に減衰
意味: 十分な出力があれば、制御系の揺らぎが自動で抑制される
```

実世界の例：「精神的な余裕があれば厳密さが柔らかくなる」

## 使用方法

### 基本操作

1. **スライダーを調整**: 左右のコントロールパネルで4つのパラメータをドラッグ
   - A（ジェネレーター）: 0.0 ~ 2.0
   - B（スタビライザー）: 0.0 ~ 2.0
   - C（戦術AI）: 0.0 ~ 2.0
   - E（環境負荷）: 0.0 ~ 1.0

2. **ロボットの変化を観察**:
   - 中央のロボットSVGが色・動き・サイズで状態を表現
   - 左下の危険度ゲージが更新
   - 右下のログに警告メッセージが表示

3. **システムログを読む**:
   - リアルタイムで各システムの異常が記録される
   - 時刻付き（HH:MM:SS.mmm形式）

### インタラクティブな実験

#### 実験1: 通常稼働
```
A: 1.0,  B: 0.5,  C: 0.5,  E: 0.0
→ ロボットが明るく浮かぶ
→ RuinScore: 0.2 (安全)
→ メッセージ: "システム安定。機体状態良好。"
```

#### 実験2: 出力不足（電力喪失）
```
A: 0.2,  B: 0.8,  C: 0.5,  E: 0.3
→ ロボットが灰色に変わる
→ RuinScore: 3.0+ (注意)
→ メッセージ: "SYSTEM DOWN" (ジェネレーター停止)
```

#### 実験3: 制御過剰（TuneOverflow）
```
A: 1.2,  B: 1.8,  C: 0.5,  E: 0.0
→ 装甲が赤くなり、パルスする
→ RuinScore: 4.0+ (危険)
→ メッセージ: "制御系が過熱状態"
```

#### 実験4: AI暴走（二重困難）
```
A: 0.8,  B: 1.6,  C: 1.7,  E: 0.2
→ 全システムが赤くフラッシュ
→ ロボット全体が震える（shake + pulse）
→ RuinScore: 5.0+ (危機)
→ ログ: "WARNING: 制御干渉により戦術AIがパニック状態(Runaway)。"
         "CRITICAL: 戦術AIが過剰防衛。ジェネレーターを強制停止(Zero)。"
```

→ **TR-1と TR-2が連鎖的に発火**

#### 実験5: 救済（光による安定化）
```
危機状態から:
C: 1.7 → 0.8（AIを冷静に）
A: 0.8 → 1.5（出力を上げる）
→ ロボットが徐々に青くなる
→ RuinScore: 5.0 → 1.5（急低下）
→ ログ: "INFO: システム安定。機体状態良好。"
```

→ **TR-3により制御が自動安定化**

| パーツ | 対応 | 色 | 動き |

## 計算式詳細

### 外部負荷の投影

```javascript
nextVA = inputA - 0.5 * inputE;  // 環境負荷で出力低下
nextVB = inputB + 0.5 * inputE;  // 環境負荷で制御上昇
nextVC = inputC + 0.5 * inputE;  // 環境負荷でAI上昇
```

### RuinScore 計算

```
calcScore(v, state, w) = w * v + (state === 2 ? 2 : (state === 1 ? 1 : 0))

rA = calcScore(vA, stateA, -1)  // A高いほど安全
rB = calcScore(vB, stateB, 1)   // B高いほど危険
rC = calcScore(vC, stateC, 1)   // C高いほど危険
rE = inputE * 2                 // 環境負荷も影響

totalRuin = max(rA, rB, rC, rE)
```

## 学習ポイント

### 初級: パラメータの効果を感じる
1. A だけを上下に動かす → エネルギー出力の影響
2. B だけを増やす → 制御が強くなると暴走の危険
3. E を増やす → すべてのシステムが悪影響

### 中級: 相互作用を理解する
1. A低 + C高 → TR-1 発火でシステムダウン
2. B高 + A低 → TR-2 発火でAI暴走
3. A高で B を調整 → TR-3 で自動安定化

### 上級: バランス運用を習得
1. E が高い環境では、すべて を低めに保つ
2. A を十分確保することが全体の安定につながる
3. C と B のバランスが心理的安定性を左右

## 応用例

- **個人の心理状態を把握**: パラメータを自分の状態に当てはめて映像化
- **ストレス下での行動予測**: 環境負荷が高い時の反応を事前にシミュレート
- **组织支援**: チームメンバーの「暴走防止」策を検討
- **メンタルケア**: コンディション（E）改善の効果を量 化
- **教育**: 複雑な心理学を直感的に学ぶ

## 注記

- このシミュレーターは心理学の簡略化モデルです
- 実際の人間行動はさらに複雑で、他の多くの要因に影響されます
- ロボットのメタファーはあくまで直感的理解を助けるものです
- 医学的または心理学的診断に代わるものではありません
- パラメータの数値は相対的な比較に使用してください
