# Mental State Insight App（メンタルステート インサイト アプリ）

**今のあなたの心の動き・性格傾向・合いやすい相手・整え方を、スライダーひとつで丸ごと分析。** 複雑な心理状態を、やさしい言葉で、視覚的にわかりやすく提示するセルフチェックアプリです。

## 概要

このアプリは、ABCモデル（自然さ・評価モード・俯瞰）を基盤に、**現在の心理状態・性格傾向・対人相性・回復方法** を統合的に診断します。

プレイヤーは：
1. **現在の状態** を15のスライダー・ラジオボタンで入力
2. 複雑な計算アルゴリズムが、相互作用を計算
3. **ひと目でわかるダッシュボード** が、状態スコア・性格パターン・おすすめ対応を表示

という体験を通じて、自分の心理状態の **客観的な把握** と **具体的な対応策** が得られます。

## 主要機能

### 1. 入力パラメータ（15項目）

#### ベース状態（ユーザーの基本的な作動）
| パラメータ | 範囲 | 説明 |
|-----------|------|------|
| **自然さ・自分らしさ (A)** | 0-100 | 素直に感じたり動いたりできる感覚。高いほどペースが保ちやすい |
| **評価モード (B)** | 0-100 | 採点・比較・気にしやすさ。高いほど社会的な適応が強い |
| **頭の中の俯瞰 (C)** | 0-100 | 考えすぎ・見すぎ・気づきやすさ。高いほど分析が深い |

#### 揺れやすさと今の出方
| パラメータ | 範囲 | 説明 |
|-----------|------|------|
| **自然さの揺れ (dA)** | 0-100 | 元気さや自分らしさの波。変動が大きいほどストレスに敏感 |
| **評価モードの揺れ (dB)** | 0-100 | 気にしやすさ・採点の波。変動が大きいほど環境反応型 |
| **俯瞰の揺れ (dC)** | 0-100 | 考え込みやすさ・見すぎの波。変動が大きいほど過敏 |
| **自然さの今の出方 (stateA)** | normal / runaway / zero | ふつう / 強く出ている / 止まり気味 |
| **評価モードの今の出方 (stateB)** | normal / runaway / zero | ふつう / 強く出ている / 止まり気味 |
| **俯瞰の今の出方 (stateC)** | normal / runaway / zero | ふつう / 強く出ている / 止まり気味 |

#### 外からかかっている負荷（4種類）
| パラメータ | 範囲 | 説明 |
|-----------|------|------|
| **体調負荷 (E0)** | 0-100 | 眠気・痛み・疲労・空腹など。高いほど自然さが低下 |
| **環境負荷 (E1)** | 0-100 | 予定・人・場所・締切の圧。高いほど盤根性能が必要 |
| **対人・社会圧 (E2)** | 0-100 | 周囲の目・比較・期待・空気。高いほど評価モードが加速 |
| **ノイズ (E3)** | 0-100 | 通知・情報量・雑音・切り替えの多さ。高いほど注意散漫化 |

#### 性格の傾向（3種類）
| パラメータ | 選択肢 | 説明 |
|-----------|--------|------|
| **評価をどこから受け取るか (bType)** | 自分基準 / 他人基準 / 半々 | 採点の向き。内向は自己基準に厳しく、外向は他評価に敏感 |
| **ものの見方のクセ (cType)** | 構造で見る / 場を整える / 自分を見すぎる / 空気を拾う | 4パターンのメタ認知スタイル |
| **対人・社会圧の刺さりやすさ (socialSensitivity)** | 低め / ふつう / 高め | HSP傾向の有無。高いほどE2の影響が増幅 |

### 2. 計算ロジック（複雑な相互作用）

#### ステップ1：内部単位への変換
```javascript
const toInternal = (value) => value / 50  // 0-100 → 0-2の内部表記に
```

#### ステップ2：基本動作計算 (v)
```
vA1 = A - 0.5 × (E0 + E1 + E3)
vB1 = B + max(0.5 × E2_adjusted, 0.5 × E3)
vC1 = C + max(0.5 × E2_adjusted, 0.5 × E3, 0.5 × E0)
```
基本状態が、外部負荷によってどう変わるかを計算。

#### ステップ3：変動計算 (d)
```
dA1 = max(dA, E0, E1, E3)
dB1 = max(dB, E2, E3)  → bType による調整
dC1 = max(dC, E2, E3, E0)  → cType による調整
```
揺れやすさ（固定的な傾向）と、環境負荷の瞬間の波を最大値で統合。

#### ステップ4：spikingエフェクト（突然の強化）
```
boostB = (deltaBand(dB1) === "spike") ? 1.35 : 1
boostC = (deltaBand(dC1) === "spike") ? 1.35 : 1
boostA = (deltaBand(dA1) === "spike") ? 1.35 : 1
```
波が「spike」（1.5以上）の時は、相互作用が1.35倍に強化。

#### ステップ5：相互作用（非線形フィードバック）
```
vA2 = vA1 + (stateC === "runaway" ? 0.9 × -g(vC1) × boostC : 0)
vB2 = vB1 + (stateA === "runaway" ? 0.9 × -g(vA1) × boostA × f(bType) : 0)
vC2 = vC1 + (stateB === "runaway" ? 0.9 × g(vB1) × boostB : 0)
```
ここで `g(x) = 0.5 × tanh(0.7 × x)` は非線形変換。一つが「runaway」だと他の要素が圧縮される。

#### ステップ6：固定的な反応パターン
```
dA2 = max(dA1, stateA === "runaway" ? 1 : 0, stateA === "zero" ? 0.5 : 0, ...)
dB2 = max(dB1, stateB === "runaway" ? 1 : 0, stateB === "zero" ? 0.5 : 0, ...)
dC2 = max(dC1, stateC === "runaway" ? 1 : 0, stateC === "zero" ? 0.5 : 0, ...)
```

#### ステップ7：状態の引き継ぎと状態遷移
```
特殊ルール:
- stateC が runaway で dC2 ≥ 1.5、vA2 ≤ 1 → stateA を "zero" に遷移
- zeroLock（E2が高い or 複数のEが高い） → 回復を抑止
- socialSensitivity が high で E2 ≥ 1.5 → stateB を runaway に
```

#### ステップ8：最終スコア計算
```
scoreA = -1 × vA2 + dA2 + stateMag[stateA2]  (low が good)
scoreB = 1 × vB2 + dB2 + stateMag[stateB2]
scoreC = 1 × vC2 + dC2 + stateMag[stateC2]
scoreE0 = E0 + E0
scoreE1 = E1 + E1
scoreE2 = adjE2 + E2
scoreE3 = E3 + E3
```

各スコアは0-6に正規化 → 最高スコアに基づいて、最終的な「負荷スコア」を2/4/5/6のいずれかに丸める。

### 3. 出力：統合ダッシュボード

#### 今の読み取り（ステータス）
```
負荷スコア 0-6 に基づけ、4段階の状態を表示：

2 以下: "落ち着きあり" (tone: "今は大きく崩れにくい状態です。")
2-4: "少し負荷あり" (tone: "無理を重ねる前に整えるのがよさそうです。")
5: "要リセット" (tone: "休息や負荷調整を優先したい状態です。")
6以上: "かなりしんどい" (tone: "まず安全と休息を確保したい状態です。")
```

#### 心の動き（Drivers ランキング）
現在の影響度TOP4 を Meter で表示：
- A: 自然さ
- B: 評価モード
- C: 俯瞰
- E0-E3: 各種外部負荷

各ドライバーに対して、その値が高い場合の注釈（note）を表示。

#### 性格診断（7パターン）
入力に基づいて、ユーザーの性格傾向を分類：

| 条件 | パターン | 説明 |
|------|---------|------|
| A が dominant & A ≥ 1.2 & B < 1.2 | 自然体クリエイター | 感覚から動ける。比較がペースを崩す |
| B が dominant & bType === internal | 理想追求タイプ | 自分の基準が高い。自己採点が厳しくなりやすい |
| B が dominant & bType === external | 評価センサータイプ | 周囲の空気をすばやく拾う。比較に巻き込まれやすい |
| C が dominant & cType === c3 | 観察・自己意識タイプ | 反応を細かく観察できるが、見すぎて疲れやすい |
| C が dominant & cType === c1 | 構造整理タイプ | 流れや関係性を整理するのが得意。休みどころを忘れやすい |
| その他 | バランス調整タイプ | 複数の側面を行き来するタイプ |

#### 相性が良い相手のタイプ
各性格パターンに対して、相性が良い相手の特徴を提示。

#### おすすめの整え方（最大4個）
drivers とその閾値に基づいて、具体的なアクションを提案：

例：
- E0（体調負荷）が高い → 「まずは休息を優先。水分・軽食・横になる・痛み対策のどれかを先に」
- E2（対人・社会圧）が高い → 「比較が入る情報をいったん止める」
- E3（ノイズ）が高い → 「通知を切って、今やることを1つだけに絞る」
- stateB === runaway → 「採点より事実メモ。『できたことを1つ』だけ書くと評価モードがゆるむ」
- stateC === runaway → 「頭の中で回し続けず、3行だけ紙やメモに出すと整理しやすい」
- stateA === zero → 「自然さを戻すために、5分で終わる小さな行動を1つ」

### 4. プリセット（4種類）

クイックプリセットで、一般的な心理状態を即座にシミュレート：

| プリセット | 説明 | 典型的なパラメータ |
|-----------|------|----------|
| **落ち着いている日** | バランスの取れた状態 | A=70, B=42, C=48（全E低め） |
| **比較で疲れている日** | E2（対人・社会圧）が高い | A=38, B=76, C=72, E2=82 |
| **考えすぎで重い日** | C（俯瞰）が runaway | A=32, B=56, C=84, stateC=runaway |
| **体力が切れている日** | E0（体調負荷）が高い、stateA=zero | A=22, E0=86, stateA=zero |

## 使用方法

### 基本的な使い方

1. **スライダー入力**
   - 左側パネルの15のスライダー・ラジオボタンで現在の状態を入力
   - 「ベース状態」 → 「揺れやすさと今の出方」 → 「外からかかっている負荷」 → 「性格の傾向」 の順 で埋めていく
   - リアルタイムで右側ダッシュボードが更新

2. **ダッシュボード確認**
   - **今の読み取り**: 負荷スコア（0-6）と状態アドバイス
   - **心の動き**: 影響度ランキング（A, B, C, E0-E3）
   - **性格診断**: あなたのパターンと特徴
   - **相性が良い性格**: パートナー選びの参考
   - **おすすめの整え方**: 具体的なアクション（最大4つ）

3. **プリセット使用**
   - 上部の「落ち着いている日」ボタンなどをクリック
   - その状態のパラメータが自動入力される
   - 異なるシナリオの比較が可能

4. **初期状態へ戻す**
   - 「初期値に戻す」ボタンをクリック

### インタラクティブな実験

#### 実験1: 「体調負荷が高い日」の影響
```
初期状態から、以下のみ調整:
- 体調負荷 (E0): 86
- 自然さの今の出方: "止まり気味"

期待結果:
- 負荷スコア: 5-6
- 自然さ: 低下
- メッセージ: 「体調負荷を優先」に

教訓: 体調が低いと、他のすべてが圧縮される
```

#### 実験2: 「対人・社会圧が高い」ときの反応
```
初期状態から:
- 評価モード (B): 76
- 対人・社会圧 (E2): 82
- bType: "外向き"
- socialSensitivity: "高め"

期待結果:
- 評価モード: runaway に遷移
- 負荷スコア: 4-5
- メッセージ: 「情報カットを優先」

教訓: 外向き性格 × 社会圧の高さ = 評価モード過熱
```

#### 実験3: 「考えすぎが始まる」プロセス
```
比較で疲れている日 プリセット から始める:
- 対人・社会圧 (E2): 82
- 俯瞰 (C): 84
- stateC: "runaway"

さらに:
- ノイズ (E3): 70 に上げる
- 評価モードの出方: "runaway" に

期待結果:
- 自然さが step-by-step で低下
- 俯瞰・評価モードが相乗クラッシュ
- 自然さ: "止まり気味" に遷移する可能性

教訓: 複数の外部負荷 × runaway 状態 = 悪性スパイラル
```

#### 実験4: 「性格の傾向による違い」
```
同じ E0=70 でも:

A. cType="構造で見る" → 整理することで対応可能
B. cType="自分を見すぎる" → さらに俯瞰が深まり、負荷増加
C. cType="空気を拾う" + socialSensitivity="高め" → 対人圧も加算
 
教訓: 同じ負荷でも、性格傾向で対応力が異なる
```

## ビジュアル デザイン

### レイアウト構成
- **左パネル（1.05fr）**: 入力フォーム（4セクション×4-6項目）
  - ベース状態（3つのスライダー）
  - 揺れやすさと今の出方（3+3スライダー+3ラジオ）
  - 外からかかっている負荷（4つのスライダー）
  - 性格傾向（3つのラジオ）
  
- **右パネル（0.95fr × sticky）**: ダッシュボード（6セクション）
  - 今の読み取り（サークルゲージ付き）
  - 心の動き（ドライバーランキング）
  - 性格診断
  - 相性診断
  - おすすめの整え方（4つのアクション）
  - 診断の見方（details/summary）

### 色彩
- **背景**: 白→灰グラデーション
- **ボタン/入力**: slate-200～900のトーン
- **アクティブ状態**: slate-900（黒）
- **パルス・警告**: スコア 5以上の時に alert カラー

### インタラクティブ要素
- **スライダー**: accent-slate-900（ミニマル）
- **ラジオボタン**: 選択時に bg-slate-900 + text-white
- **メーター**: width に基づいて 0-100% まで充填
- **サークルゲージ**: 負荷スコアを0-6に正規化して圓周アニメーション

## 心理学的背景

このアプリは、以下の概念を統合しています：

### ABCモデル（拡張版）
- **A（自然さ）**: 自分らしさ、inner directed
- **B（評価モード）**: 採点・比較、社会的順応
- **C（俯瞰）**: メタ認知、自己反映

### 複雑系・非線形フィードバック
- 一つの要素が「runaway」（過剰）だと、他の要素が圧縮される
- 外部負荷が高いほど、基本状態の変動が大きくなる
- 特定のパターン（例：stateC runaway + stateA zero）で状態遷移が固定化

### 環境負荷の4レイヤー
1. **E0（体調）**: 身体的制約
2. **E1（環境）**: 物理的・時間的負荷
3. **E2（対人・社会圧）**: 他者の期待・比較
4. **E3（ノイズ）**: 情報・刺激・集中喪失

### HSP（高刺激処理感受性）の概念
- socialSensitivity パラメータで、人によって環境影響が異なることを表現

## 応用例

- **自己理解**: 自分の反応パターンの客観化
- **マインドフルネス**: 今の状態を認知して対応
- **チームビルディング**: メンバーの視点・負荷を理解
- **心理学教育**: 複雑なシステムの相互作用を直感的に学ぶ
- **ウェルネス推奨**: 個人のパターンに基づいた提案

## 注記

- このアプリは **自己認識ツール** であり、医学的診断ではありません
- スコア・診断は **相対的な参考**です（絶対的な判断ではない）
- 実際の心理状態はさらに個人差・環境差が大きいため、参考値です
- セルフチェックアプリとして活用し、必要に応じて専門家の協力を求めてください
