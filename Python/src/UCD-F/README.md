# UCD-F_ABC Python Core Implementation

## 1. システム概要
本プロジェクトは、制約下動的状態遷移システム「Universal Constrained Dynamics Framework (UCD-F)」のコアロジックをPythonで実装したものです[cite: 1]。
以下の数理モデルに基づき、エンティティの状態遷移と環境エントロピーの相互作用を制御します[cite: 1]。

$$System = (S, A, C, B, D, M, \epsilon, V, T, N)$$[cite: 1]

## 2. 主要機能
*   **魂のビットパッキング (Bit-packed SoA)**: 32bit整数のビット演算により、メモリ使用量を最小化しつつL1/L2キャッシュヒット率を極大化します[cite: 1]。
*   **VIF/ABC 統合遷移モデル**: 「速度(Velocity)・強度(Intensity)・疲労(Fatigue)」の三要素を用いた、動的なABC状態遷移を実装しています[cite: 1]。
*   **現実エントロピーの収穫 (Entropy Harvesting)**: デバイス負荷や外部環境ノイズをシミュレーションの入力として動的に正規化・注入します[cite: 1]。
*   **Zero-Allocation 設計**: 実行中の動的なメモリ確保を排除し、初期化時に確保した固定長プールのみで演算を行うことで、GC（ガベージコレクション）によるスパイクを抑止します[cite: 1]。

## 3. メモリレイアウト（32bitパッキング構造）
各エンティティの状態は、`numpy.uint32` 配列の1要素内に以下の仕様で格納されます[cite: 1]。

| ビット範囲 | 識別子 | 内容 |
| :--- | :--- | :--- |
| `[0-1]` | `vA` | 指標Aの強度 (2bit)[cite: 1] |
| `[2-3]` | `vB` | 指標Bの強度 (2bit)[cite: 1] |
| `[4-5]` | `vC` | 指標Cの強度 (2bit)[cite: 1] |
| `[6-11]` | `dA` | 変動ΔA (6bit, 符号付)[cite: 1] |
| `[12-17]` | `dB` | 変動ΔB (6bit, 符号付)[cite: 1] |
| `[18-23]` | `dC` | 変動ΔC (6bit, 符号付)[cite: 1] |
| `[24-26]` | `State` | 特殊状態フラグ (3bit)[cite: 1] |
| `[27-29]` | `RuinScore` | 累積破綻値 (3bit)[cite: 1] |
| `[30-31]` | `Reserve` | 拡張用予約領域 (2bit)[cite: 1] |



## 4. 実行の原則
1.  **完全決定論 (Determinism)**: 演算から浮動小数点を排除し、固定小数点演算（または整数演算）を用いることで、ハードウェアに依存しない一貫した結果を保証します[cite: 1]。
2.  **死骸の地形化**: `RuinScore` が上限に達し「破綻」した個体は、動的計算から除外され、環境要因（エントロピーソース）として再定義されます[cite: 1]。
3.  **局所時間拡張 (Time Dilation)**: 変動強度が低い領域の更新頻度を動的に下げることで、計算リソースをカオス領域へ集中させます[cite: 1]。

## 5. 依存関係
*   `numpy`: 高速なSoA（Structure of Arrays）演算のために必須です[cite: 1]。
